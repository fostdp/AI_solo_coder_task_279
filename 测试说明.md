### 4. 🔬 交互测试区域（手动测试）
包含可视化的拖拽测试区域，可以：
- 点击按钮生成1个或5个测试物品
- 支持鼠标和触摸屏拖拽
- 实时显示拖拽状态
- 放入垃圾桶时有视觉反馈和动画
- 边界检测防止物品移出可视区域

### 5. 📜 拖拽与页面滑动冲突测试
专门解决移动端常见的手势冲突问题：

| 机制 | 说明 |
|-----|------|
| 方向检测 | 水平移动 -> 拖拽；垂直移动 -> 滚动 |
| 长按阈值 | 长按200ms后进入拖拽模式 |
| 方向锁定 | 拖拽开始后锁定移动方向 |
| 边界检测 | 防止物体移出父容器 |
| 事件冒泡 | 正确阻止事件冒泡 |
| 滚动恢复 | 拖拽结束后恢复滚动功能 |

## 测试使用方法

### 快速开始
1. 在浏览器中打开 `test.html`
2. 点击 **"▶️ 运行所有测试"** 按钮执行全套测试
3. 查看测试统计面板和各测试区域的结果

### 单独运行测试
- **分类规则测试**: 点击 **"📋 分类规则测试"**
- **物体生成测试**: 点击 **"🎲 生成测试"**
- **冲突测试**: 点击 **"开始冲突测试"**

### 交互测试（手动）
1. 在交互测试区域点击 **"生成5个"**
2. 用鼠标或手指拖拽物品
3. 拖到对应垃圾桶上观察反馈
4. 尝试在滚动区域垂直/水平拖拽，观察行为

## 关键测试点详解

### 分类规则测试逻辑
```javascript
// 测试上海标准的特殊分类
testGame.currentRegion = 'shanghai';
let correctType = testGame.regionStandards['shanghai'].special['大骨头'];
// 期望: 'other' 即其他垃圾
```

### 物体生成边界检测
```javascript
// 生成时确保不会超出边界
x = Math.max(0, Math.min(x, containerWidth - itemWidth));
y = Math.max(0, Math.min(y, containerHeight - itemHeight));
```

### 拖拽冲突解决方案
```javascript
// 触摸事件处理
touchStart: 记录起始位置和时间
touchMove: 
  - 计算移动方向
  - 水平位移大：拖拽模式
  - 垂直位移大：滚动模式
  - 超过长按阈值：强制进入拖拽
```

## 常见问题

### Q: 测试全部失败怎么办？
A: 检查浏览器控制台是否有JS错误，确认 `game.js` 正确加载

### Q: 拖拽不流畅？
A: 这是正常现象！测试页面故意保留了原生滚动行为，用于验证冲突处理逻辑

### Q: 如何添加新测试用例？
A: 在对应的 `testCases` 数组中添加对象，包含 `name`、`test` 函数即可
